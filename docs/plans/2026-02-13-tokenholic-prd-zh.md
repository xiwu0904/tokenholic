# Tokenholic - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
**Tokenholic**

### 1.2 愿景
一款AI赋能的销售工具平台，帮助阿里云销售团队发现并赢取企业客户的LLM/MaaS（模型即服务）Token商机。

### 1.3 问题陈述
阿里云销售团队在快速增长的LLM/AI市场面临重大挑战：

- **信息透明度不足**：难以了解客户在其业务中如何使用AI/LLM
- **竞争情报缺失**：难以识别哪些工作负载运行在竞争对手平台（火山引擎、百度智能云、Azure等）而非阿里云
- **方案复杂度高**：难以针对每个客户独特的AI用例 articulate 合适的解决方案
- **准备不充分**：缺乏有效练习和准备复杂技术销售对话的能力

### 1.4 解决方案
Tokenholic提供四项AI赋能的核心能力：

| 能力 | 描述 |
|------|------|
| **Token战场图生成器** | 自动研究客户并可视化其AI Token消耗全景，揭示竞争机会 |
| **方案推荐引擎** | 基于客户用例推荐定制化阿里云解决方案，由RAG知识库驱动 |
| **销售角色扮演模拟器** | AI驱动的练习环境，模拟多种场景下的客户对话 |
| **迭代优化引擎** | 捕获客户反馈，通过版本迭代演进解决方案 |

## 2. 目标用户

### 2.1 主要角色

#### 销售代表
- **职责**：一线客户对接
- **核心需求**：快速客户调研、有说服力的演示、对话准备
- **主要使用功能**：Token战场图、角色扮演模拟器

#### 销售经理/负责人
- **职责**：跨多客户的战略客户规划
- **核心需求**：组合层面的机会可视化、团队赋能
- **主要使用功能**：Token战场图（汇总视图）、方案历史

#### 售前工程师/解决方案架构师
- **职责**：技术方案设计与验证
- **核心需求**：准确的技术推荐、竞争差异化
- **主要使用功能**：方案推荐引擎、迭代优化引擎

### 2.2 用户体验原则
- 针对每个角色定制化视图和工作流
- 从第一天起支持中英双语界面
- 同时支持桌面端和移动端的响应式设计
- 直观、视觉冲击力强的UI，让复杂数据易于理解

## 3. 核心功能

### 3.1 Token战场图生成器

#### 3.1.1 概述
生成客户AI/LLM Token消耗的全景可视化图，类似于参考文档"Token业务白板图"。

#### 3.1.2 可视化组件
- **业务用例映射**：剧本创作、图像生成、视频制作、配音音效、剪辑包装等
- **各用例Token消耗**：估算的Token量级（以亿为单位）
- **云厂商分布**：按厂商颜色编码（阿里云、火山引擎、百度智能云、海外模型、自建IDC）
- **模型归属**：每个用例使用的具体模型
- **机会指标**：高亮阿里云可抢占市场份额的领域
- **增长预测**：预估增长率和未来Token量级

#### 3.1.3 交互模式
- **交互式仪表盘**：点击下钻、按用例/厂商筛选、缩放
- **轻度编辑**：销售人员可添加备注、调整估算、展示前自定义
- **导出功能**：生成可分享的客户演示格式

#### 3.1.4 数据来源

**一期（公开数据）**：
- 公司官网和技术博客
- 新闻报道和新闻稿
- 招聘信息（反映AI/ML招聘需求）
- GitHub仓库和开源活动
- 行业报告和分析师覆盖

**后续阶段**：
- 内部CRM数据集成
- 行业基准数据库
- 私有数据源连接器

**始终可用**：
- 手动输入界面，供销售添加已知信息

### 3.2 方案推荐引擎

#### 3.2.1 概述
根据客户识别的用例和需求，提供定制化的阿里云解决方案推荐。

#### 3.2.2 技术架构
- **RAG系统**：基于检索增强生成技术，从精选知识库中获取信息
- **可溯源回答**：所有推荐均引用来源文档
- **用例映射**：将客户需求映射到阿里云产品和服务

#### 3.2.3 知识库内容

**一期（公开来源）**：
- 阿里云官方文档
- 产品页面和功能列表
- 已发布案例研究
- 定价信息（公开档位）
- 技术博客文章

**后续阶段**：
- 内部解决方案架构
- 竞品迁移指南
- ROI计算器和TCO模型
- 客户成功案例（获授权）

#### 3.2.4 推荐输出
- 推荐的阿里云产品/服务
- 架构图（适用时）
- 相对当前/竞品方案的关键差异化优势
- 实施考虑事项
- 相关案例研究或参考资料

---

### 3.3 销售角色扮演模拟器

#### 3.3.1 概述
AI驱动的模拟环境，销售团队成员在此练习客户对话。

#### 3.3.2 可选场景

| 场景 | 描述 | 难度等级 |
|------|------|----------|
| **客户发现** | 了解客户的AI需求和痛点 | 简单 / 中等 / 困难 |
| **技术深聊** | 回答关于阿里云LLM产品的详细问题 | 简单 / 中等 / 困难 |
| **异议处理** | 应对常见反对意见（价格、安全、供应商锁定） | 简单 / 中等 / 困难 |
| **高管汇报** | 面向C-level的业务价值导向演示 | 简单 / 中等 / 困难 |

#### 3.3.3 功能特性
- **用户自选场景**：销售选择要练习的场景
- **可定制客户人设**：行业、公司规模、技术成熟度
- **实时反馈**：AI评估回答并提供改进建议
- **进度追踪**：追踪已完成的场景和表现趋势

#### 3.3.4 AI行为
- 模拟真实客户人设，带有适当的怀疑态度
- 根据销售回答提出追问
- 在适当时机提出常见异议
- 根据所选难度调整行为

---

### 3.4 迭代优化引擎

#### 3.4.1 概述
允许销售捕获客户反馈并迭代优化方案，直至可成交。

#### 3.4.2 工作流程
1. 销售向客户展示初步方案
2. 客户提供反馈（顾虑、需求、问题）
3. 销售以自由文本形式输入反馈
4. AI根据反馈重新生成优化方案
5. 重复直至方案被接受

#### 3.4.3 功能特性
- **简单文本输入**：自然语言反馈录入
- **版本历史**：追踪所有方案版本及变更内容
- **差异对比**：查看版本间变更
- **回滚功能**：可恢复至之前版本

#### 3.4.4 一期范围排除项
- 无结构化反馈表单
- 无多人协作功能
- 无赢单/输单追踪

### 4.2 前端技术栈

| 组件 | 技术 | 用途 |
|------|------|------|
| 框架 | React 18+ | 组件化UI |
| 构建工具 | Vite | 快速开发和构建 |
| 样式 | TailwindCSS + Shadcn/UI | 现代响应式设计 |
| 状态管理 | Zustand 或 React Query | 客户端状态 |
| 可视化 | D3.js / ECharts | Token战场图渲染 |
| 国际化 | react-i18next | 双语支持 |

### 4.3 后端技术栈

| 组件 | 技术 | 用途 |
|------|------|------|
| API框架 | FastAPI | REST API端点 |
| 智能体框架 | AgentScope | 多智能体编排 |
| LLM访问 | 百炼SDK | 模型推理 |
| 向量存储 | FAISS / Milvus | RAG嵌入 |
| 数据库 | PostgreSQL | 持久化存储 |
| 缓存 | Redis | 会话和查询缓存 |

### 4.4 多智能体架构

**按功能组织**：每个主要功能有独立的智能体系统，便于模块化和独立扩展。

#### Token地图智能体系统
- **研究智能体**：收集客户公开数据
- **分析智能体**：处理和结构化发现
- **可视化智能体**：生成地图数据结构

#### 方案推荐智能体系统
- **查询智能体**：处理用户问题
- **检索智能体**：搜索知识库
- **合成智能体**：组织推荐内容

#### 角色扮演智能体系统
- **人设智能体**：模拟客户行为
- **评估智能体**：评估销售回答
- **反馈智能体**：提供改进建议

### 5.2 持久化策略

| 数据类型 | 存储 | 保留期限 |

## 6. 一期范围

### 6.1 包含范围

| 功能 | 范围 |
|------|------|
| Token战场图 | 完整功能，使用公开数据源 |
| 方案推荐引擎 | 基于公开阿里云文档的RAG |
| 角色扮演模拟器 | 全部4个场景，3个难度等级 |

### 6.2 一期不包含

| 功能 | 原因 | 目标阶段 |
|------|------|----------|
| 用户认证 | 先做演示模式 | 二期 |

| 赢单/输单追踪 | 演示阶段非关键 | 三期 |
| 协作功能 | 复杂度高 | 三期 |
| 高级分析 | 需要先积累使用数据 | 三期 |

### 6.3 一期成功标准

- [ ] 输入公司名称即可生成Token战场图
- [ ] 针对识别的用例产出相关方案推荐
- [ ] 完成全部4种场景类型的角色扮演会话
- [ ] 方案可迭代3轮以上反馈循环
- [ ] UI在桌面端和移动端正确渲染
- [ ] 系统支持中英文响应
